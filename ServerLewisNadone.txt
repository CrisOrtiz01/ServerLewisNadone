# ServerLewisNadone - Documento Técnico y Explicativo

1. Lore y Ambientación

El servidor está ambientado en un pueblo fundado por dos campeones legendarios, que transmitieron sus conocimientos a generaciones futuras. Hoy, el pueblo es guiado por 9 líderes, quienes reciben a los jugadores (extranjeros que buscan ser los proximos campeones del pueblo) y los introducen en una constante búsqueda de poder y superación. El lore se apoya en la interacción con NPCs, misiones y progresión RPG.

2. Plugins Principales

- **EssentialsX**: Comandos básicos, homes, kits, warps.
- **LuckPerms**: Gestión avanzada de permisos y rangos.
- **GeyserMC + Floodgate**: Compatibilidad Bedrock ↔ Java.
- **Citizens2**: Creación de NPCs interactivos.
- **Quests**: Sistema flexible de misiones, integración con Citizens.
- **MythicMobs**: Creación y gestión de mobs custom, reemplazo de mobs vanilla.
- **MMOItems/MMOCore**: Objetos RPG, clases, profesiones, habilidades (si está disponible).
- **Lands/FactionsUUID**: Gestión de territorios y facciones.
- **HolographicDisplays**: Texto flotante para ambientación.
- **TAB**: Customización del tablist.
- **ItemsAdder/Oraxen**: Texturas, bloques y objetos custom.
- **CoreProtect**: Protección y rollback.
- **WorldGuard + WorldEdit**: Protección de regiones y utilidades de construcción.
- **ClearLag**: Optimización de entidades.

## 5. NPCs y Misiones (Citizens2 + Quests)

### NPCs Principales

#### 1. Erika la Cazadora
- **Función**: Proveedora de animales al pueblo
- **Misiones**: Caza de mobs hostiles y neutrales
- **Recompensa**: Habilidad de Fuerza (niveles 1-10)
- **Progresión**:
  - **Fuerza 1**: Matar 20 zombies/esqueletos + 50 tokens
  - **Fuerza 5**: Matar mobs del Nether + 120 tokens
  - **Fuerza 10**: Matar los 3 jefes principales + 300 tokens

#### 2. Marta, La Guardiana
- **Función**: Seguridad del pueblo
- **Misiones**: Caza de Creepers y recolección de materiales defensivos
- **Recompensa**: Habilidad de Resistencia (niveles 1-3)
- **Progresión**:
  - **Resistencia 1**: Eliminar 15 creepers + 32 tokens
  - **Resistencia 3**: Eliminar 25 creepers + 64 tokens

#### 3. Sae, la Sanadora
- **Función**: Curación y sanación
- **Misiones**: Proteger aldeanos y recolectar ítems curativos
- **Recompensa**: Habilidad de Salud Extra (niveles 1-5)
- **Progresión**:
  - **Salud Extra 1**: Matar 5 saqueadores + 60 tokens
  - **Salud Extra 5**: Matar 25 saqueadores + 140 tokens

#### 4. Lia, la Mensajera
- **Función**: Entrega de mensajes, velocista del pueblo
- **Misiones**: Carreras y pruebas de velocidad
- **Recompensa**: Habilidad de Velocidad (niveles 1-3)
- **Progresión**:
  - **Velocidad 1**: Completar carrera básica + 32 tokens
  - **Velocidad 3**: Completar carrera experta + 128 tokens

#### 5. Ícaro, el Acróbata
- **Función**: Entretenimiento, acrobacias en el circo
- **Misiones**: Parkour y desafíos de agilidad
- **Recompensa**: Habilidad de Salto Aumentado (niveles 1-4)

## 7. Configuración de Archivos y Análisis de Sistema

### MythicMobs

- **Config-Spawning.yml**:
  - DisableVanillaSpawns: false (Crucial para permitir el reemplazo de mobs)
  - SpawnRadiusPerPlayer: 64 (Radio de spawn alrededor del jugador)
  - LimitMultiplier: 1.2 (20% más de mobs que vanilla)
  - ReplaceObeysCap: false (El reemplazo no obedece límite de mobs)

- **Config-Mobs.yml**:
  - DefaultDropMethod: VANILLA (Sistema de drops vanilla)
  - DoLootsplosionByDefault: false (No explosión de loot por defecto)
  - MinimumDamagePercentForDrops: 1% (Requiere poco daño para drops)

- **RandomSpawns**:
  - Archivo ejemplificativo para spawns aleatorios personalizados
  - No usado actualmente pero disponible para implementación futura

### Sistema de Traducciones (Quests)

- Dos archivos de lenguaje: es-MX y es-ES
- Traducciones completas para toda la interfaz de misiones
- Mensajes personalizados para cada tipo de objetivo y recompensa

## 8. Análisis de Reemplazo de Mobs

El sistema utiliza `SpawnOverride` en todos los mobs custom para reemplazar sus equivalentes vanilla:

- **Mecanismo**: Cuando un mob vanilla intenta spawnear, MythicMobs lo intercepta y lo reemplaza
- **Condiciones**: La mayoría usa `always` como condición (reemplazo incondicional)
- **Prioridades**: Algunos mobs tienen diferentes prioridades (1-2) para control de spawn
- **Requisitos**: DisableVanillaSpawns debe estar en false para funcionar

### Posibles Problemas de Spawn Detectados:

1. Algunos archivos YAML pueden tener problemas de indentación
2. Algunos mobs pueden tener configuraciones de spawn contradictorias
3. Conflictos potenciales con otros plugins de gestión de mobs
4. La configuración actual no contempla tasas de spawn diferenciadas por bioma para todos los mobs

## 9. Recomendaciones de Mejora

1. **Expandir Sistema de Drops**:
   - Implementar sistema de drop tables más complejas
   - Añadir ítems con efectos especiales como recompensas raras

2. **Balanceo de Dificultad**:
   - Ajustar atributos de mobs según zonas y progresión
   - Crear variantes "élite" con menor tasa de spawn pero mayor recompensa

3. **Integración con Otros Plugins**:
   - Conectar con MMOItems para ítems más complejos
   - Utilizar ModelEngine para modelos 3D de mobs únicos

4. **Optimización de Spawn**:
   - Configurar spawn rates específicos por bioma y dimensión
   - Limitar spawns de jefes menores a zonas específicas

5. **Sistema de Misiones Avanzado**:
   - Crear cadenas de misiones con narrativa
   - Implementar misiones grupales que requieran cooperación

## 10. Conclusión

El servidor ServerLewisNadone implementa un sistema RPG detallado con:
- 21+ mobs custom totalmente configurados con atributos, habilidades y loot
- 9 NPCs con progresión de misiones y recompensas
- Sistema de ítems custom para progresión
- Reemplazo completo del ecosistema de mobs vanilla
- Sistema de misiones con múltiples niveles de dificultad

La estructura actual proporciona una base sólida, pero requiere pruebas y ajustes constantes para balancear la dificultad y asegurar que el sistema de spawn de mobs funcione correctamente en todas las dimensiones y biomas.
